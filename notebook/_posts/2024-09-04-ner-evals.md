<span class="firstcharacter">T</span>he scope of  large language model (LLM) applications today covers not only generation tasks such as question-answering or summarization, but also classical NLP tasks such as dependency parsing and named entity recognition.
In this blog post, I want to revisit how we usually evaluate them on NER tasks.

Because most LLMs are causal decoders, the common interface for using them in NER is through *prompting*.
In its basic form, we prompt a language model by passing a template with the text-in-question and a set of potential named-entities.
Several works such as [spacy-llm](https://github.com/explosion/spacy-llm) and PromptNER ([Ashok and Lipton, 2023](https://arxiv.org/pdf/2305.15444)) optimize this template by adding few-shot demonstrations or label definitions:


<!-- insert image -->

In return, the language model gives us a response that we parse to obtain the entities.
The challenge here is in shaping the output so that our parser can obtain these texts.
We can write the most robust parser we can think of, but the output of a language model will always be non-deterministic.
To get the accuracy, we compare the parsed LLM output to the gold reference data.
For the purposes of this blog post, I call this evaluation approach **generative**.
